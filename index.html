环境搭建—JDK11与Tomcat9
一、下载 JDK、Tomcat 下载地址详见 我的博客，找当相应的软件点击后面的立即下载 二、配置​ 因为JDK11后，就没有JRE，所以有些许不一样 ​ 这里默认JDK放在了D:\Other\Application\jdk-11.0.5 ​ Tomcat放在了D:\Other\Application\apache-tomcat-9.0.30 在系统变量里添加JAVA_HOME：D:\Other\Application\jdk-11.0.5 在系统变量里添加CATALINA_HOME：D:\Other\Application\apache-tomcat-9.0.30 在系统变量里的Path新建 %CATALINA_HOME%\lib %CATALINA_HOME%\bin %JAVA_HOME%\bin （可选）在cmd或者Powershell输入java -version看是否显示版本号，以验证java环境配置是否正确 在资源管理器打开Tomcat的安装目录下的bin文件夹 按住shift再点击鼠标右键，点击 在此处打开Powershell窗口 输入service install Tomcat9 三、运行 在 此电脑 右键—>选择管理—>选择左边的服务与应用服务 选择服务—>找到Apache Tomcat—>右键 启动（想要停止可以选择停止） 在浏览器输入http://localhost:8080/看是否可以访问 四、常见问题1、端口冲突 找到Toncat目录下的conf文件夹下的server.xml Shutdown端口 Server port（默认为8005端口） http访问端口 Connector port（默认为8080端口） AJP协议访问端口 Connector port（默认为8009端口） 2、更改用户名 找到Toncat目录下的conf文件夹下的tomcat-users 将下面的注释代码换成下面的代码，用户名与密码都是root <role rolename="root"/> <user username="root" password="root" roles="manager-gui,admin-gui"/> 3、Tomcat在IDEA中控制台乱码 ​ 在apache-tomcat-9.0.16\conf\logging.properties中将java.util.logging.ConsoleHandler.encoding = UTF-8注释掉
