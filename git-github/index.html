一些Git的quick start与入门知识：配置信息，仓库的创建、提交、删除、重命名、查看日志、修改提交描述、给Git命令起别名、分支等

1.初入手该做的事情——配置个人（作者）信息

更改全局的信息

#配置全局的作者邮箱
git config --global user.email "qujihan@163.com"
#配置全局的作者名字
git config --global user.name "Qu"
#可以在C:\Users\${username}\.gitconfig 可以查看用户的全局设置

更改某个仓库的信息

#配置项目作者邮箱
git config user.email "qujihan@163.com"
#配置项目作者姓名
git config user.name "Qu"
#查看配置信息(信息一般在git目录下的config文件中)
code git/config

创建一个新的仓库

#新建
git init

版本库忽略文件的配置

#创建一堆文件
touch a.txt b.txt c.txt d.txt e.txt f.txt
#创建文件夹并且在其中创建文件
mkdir test && touch test/qu.txt
#创建忽略文件
touch .gitignore

忽略文件的配置：

  1. *.txt ：忽略所有的txt文件

  2. a.txt ：忽略a.txt

  3. *.txt !a.txt ：忽略除了a.txt以外的所有txt

  4. /test ：忽略test文件夹里面的内容

当且仅当文件夹里的所有文件都被忽略的时候，使用git status才不显示这个文件

2.Git使用中——接手新项目

接手一个旧的仓库

#从github等网站克隆一个仓库（后面是克隆的地址）
git clone https://github.com/Qujihan/OperatingSystem_Algorithm.git

从版本库中删除文件

#这样删除的话，本地和版本库中的a.cpp都删除了
git rm a.cpp
#这样可以从版本库中删除，但是本地是不删除的
git rm -cached a.cpp

更改文件的名字

#讲xxx.cpp改成aaa.cpp
git mv xxx.cpp aaa.cpp

Git的流水线操作

#先创建一个文件
touch a.cpp
#将这个文件放到小推车里
git add a.cpp
#也可以全部添加进去
git add .
##将文件送到仓库
git commit -m "这里写一下提交情况"

查看日志

#查看所有的日志信息
git log
#查看最近一(或者n次)次的信息
git log -1（n）
#查看更加详细的信息
git log -p
#查看更加简短的信息
git log -oneline
#查看文件发生的变化
git log --name-only
#查看文件变化更多的信息（添加删除之类的）
git log --name-status

修改提交的描述

#调用vim对提交信息进行修改
git commit --amend

3.Git的分支使用

分支

#查看分支(绿色标志是当前分支)
git branch
#查看分支（包含远程分支:橘黄色）
git branch -a
#创建qu分支
git branch qu
#切换到qu分支
git checkout qu
#创建并切换分支qu
git checkout -b qu
#分支的合并（将qu分支合并到当前的分支中）
git merge qu

touch demo.cpp后，在qu分支commit后，切换到master分支再ls是没有的。

当然，有时候需要删除。删除不能盲目删除，需要看看有没有被merge。

#查看已经merge的分支
git branch --merge
#查看没有merge的分支
git branch --no--merge
#删除分支qu(d-delete)
git branch -d qu
#强制删除分支qu（qu可能没有合并之类的）
git branch -d qu

分支之冲突

两个分支merge到了master分支的时候，如果有冲突，需要在master分支中手动修改

分支之暂存

场景：我在ask分支开发，但是bbs分支出现了毛病，我需要到bbs分支修改，但是ask分支的内容没有写完，提交不太好。但是不提交还不能切换分支。

这时候就应该用到暂存功能

#暂存
git stash
#查看所有的暂存
git stash list
#恢复暂存
git stash apply
#恢复了，吧暂存区的内容删除（删除stash@{0}，名字在list中可以查看）
git stash drop stash@{0}
#使用pop就可以恢复+删除
git stash pop

新建的文件不会提交的暂存区，只有add后或者修改的文件才可以提交的暂存区

4.Git的进阶使用

删除暂存中已经添加的文件/从版本库中恢复

#从暂存区中删除（第一次放入暂存区）
git rm --cached a.cpp
#撤销在暂存中的文件（第二次或者以后的删除）
git reset HEAD a.cpp
#将本地文件恢复到版本库中的版本
git checkout -- a.cpp

给Git命令起个别名

#给commit起了一个c的别名
git config --global alias.c commit

起的别名都可以在gitconfig配置文件中看到

可以看出，Git 只是简单地将别名替换为对应的命令。 然而，你可能想要执行外部命令，而不是一个 Git 子命令。 如果是那样的话，可以在命令前面加入 ! 符号，如：

git config --global alias.hexonet '!hexo g && hexo d'

Tag标签

#查看所有的标签
git tag
#打标签（v1.0版本）
git tag v1.0

生成ZIP压缩包

git archive master --prefix='demo/' --forma=zip >demo.zip

5.Github的使用

SSH连接

#生成ssh key
ssh-keygen -t rsa -C "qujihan@163.com"
#查看公钥
cat ~/.ssh/id_rsa.pub

.pub 文件是公钥，另一个则是私钥，将文件复制到github/gitee的ssh配置里面就可以了，如果重新生成了Key，可能连接不上，这时候需要删除git家目录下的known_hosts文件里的内 容（known_hosts与id_rsa.pub在同一个目录下）

本地与远程关联

#将本地文件与服务器的仓库进行关联
git remote add origin https://github.com/Qujihan/gitStudy.git
#将文件推送到master分支
git push -u origin master
#将本地的分支qu与远程进行创建
git push --set-upstream origin qu

将远程分支下载到本地分支

#将远程的qu分支下载到本地的han分支
git pull origin qu:han

将远程分支删除

#删除远程分支qu
git push origin --delete qu

6.日常中Git出现的问题以及解决方案

  1. warning: LF will be replaced by CRLF in
    • 出现原因：windows中的换行符为 CRLF， 而在linux下的换行符为LF，所以在执行add . 时出现提示
    • 解决方法：git config –global core.autocrlf false

7.Github中如何搜索开源项目

在GitHub中，每一个项目都是由项目名(name)、描述(description)、源码与README.md组成，所以搜索的时候要按照这四部分搜索。当然还有一些限制因素，比如语言、时间、星星数量、Fork等因素

  • in:name xxx // 按照项目名搜索
  • in:readme xxx // 按照README搜索
  • in:description xxx // 按照description搜索
  • stars:>xxx // stars数大于xxx
  • forks:>3000 // forks数大于xxx
  • language:xxx // 编程语言是xxx
  • pushed:>YYYY-MM-DD // 最后更新时间大于YYYY-MM-DD
